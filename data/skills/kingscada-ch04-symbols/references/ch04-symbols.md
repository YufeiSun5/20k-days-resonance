第四章 精灵图

## 4.1      概述

所谓精灵图就是在外观上类似于组合图素，但内嵌了丰富的动画连接和逻辑控制，工程人员只需将其从精灵图库中调出来放在画面上，再做少量的修改，就能完成复杂的控制功能。

精灵图可以由图素对象、自定义变量、自定义属性及动画处理四部分组成。组成精灵图的图素对象可以是直线、折线、矩形、椭圆等简单图素或者他们的组合图，也可以是精灵图对象，即精灵图的开发是可以嵌套的。

## 4.2      创建精灵图

在KingSCADA画面编辑器中执行“文件” →“新建” →“精灵图”命令，弹出精灵图开发界面，如图所示：

图4-1 精灵图开发界面

精灵图开发界面与图形模型开发界面很相似，由菜单栏、工具栏、工具箱、图形编辑区、属性窗口等组成，操作方法也基本相似，唯一一点不同的是，图形模型具有传播性，当对工程图形模型进行修改后，工程中该模型的所有对象都能自动做出相应修改，也就是说对工程图形模型的修改能够传播到它的对象中，而精灵图不具有这种特性，精灵图被引用到画面中后，在画面中进行修改。

## 4.3      修改精灵图

对于创建完的精灵图，可以对其进行修改，方法如下：

在画面编辑器中执行“文件”→“打开”→“精灵图”命令打开精灵图库，选择预修改的精灵图，单击“确定”按钮后，在弹出的精灵图界面上修改精灵图。

## 4.4      精灵图脚本

精灵图脚本是指与精灵图显示与否有关的命令语言程序，精灵图命令语言是在图形模型开发界面中定义的，定义方法如下：

在KingSCADA精灵图开发界面中单击“动画连接”窗口中的  图标，弹出二级菜单，如图所示：

图4-15 精灵图脚本菜单

精灵图脚本分为精灵图打开时、存在时和关闭时的命令语言。

**打开时：**打开或激活当前精灵图，或精灵图由隐藏变为显示时执行一次。

**存在时：**精灵图在显示时，或精灵图由隐藏变为显示时周期性执行，在“每…毫秒”编辑框中设置执行周期，默认是3000毫秒。

**关闭时：**精灵图由当前激活状态变为隐藏或关闭时执行一次。

## 4.5      精灵图库

KingSCADA精灵图库用来存储系统自带的精灵图和用户开发的精灵图，它存在于软件安装目录下（即Glib目录下）。

在KingSCADA画面编辑器中执行“文件”→“打开”→“精灵图”命令打开精灵图库，如图所示：

图4-16 精灵图库

在该对话框中包含很多目录如：Arrows、BackGround、Fans、Meter等，每个目录下包括很多精灵图，有些精灵图是系统自带的，有些精灵图是用户自己开发的。选中某个目录单击鼠标右键，弹出右键菜单，如图所示：

图4-17 精灵图库右键菜单

菜单命令及功能描述如下：

|  |  |
| --- | --- |
| **命令** | **功能描述** |
| 新建目录 | 使用此命令或工具栏中的按钮，在精灵图目录区中建立目录，该目录可以指定保存的路径 |
| 添加已存在的库 | 使用此命令或工具栏中的按钮，可以将其他路径下（或者是其他计算机下）的精灵图库添加到当前图库中，精灵图库文件的扩展名是.kgn |

在图4-16中选择任一目录，在该窗口右侧显示出该目录下所包含的精灵图，选中任一精灵图，弹出右键菜单，如图所示：

## 4.6      精灵图应用

精灵图库中的精灵图主要应用在画面中，在画面编辑器中打开任一画面，执行“对象”菜单中的“精灵图”命令或者单击工具栏中的按钮，在打开的精灵图库中选择需要的精灵图后，单击“确定”按钮即可将精灵图调到画面中来。精灵图的使用过程同图形模型实例化过程，请详见第13章模型。

本机想得到另外一台机器上创建的精灵图，可以按照如下步骤加载精灵图：

步骤一：

打开精灵图，选择要加载的精灵图，则精灵图文件夹所在的路径会显示在“路径名”后的文本框中（如下图所示），将相应路径下的文件（\*.kgn）找到并复制。

步骤二：

将步骤一复制的精灵图文件拷贝到本机上，例如C:\Program
Files\KingSCADA\bin\Glib，然后在图形开发系统下，打开精灵图，点击“新建”，在弹出的对话框中添加精灵图，确定后，就把另一台机器的精灵图加载到本机上了（如图下图所示）。
