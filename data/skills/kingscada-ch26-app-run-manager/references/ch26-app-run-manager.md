第二十六章 应用运行管理器

## 26.1概述

应用运行管理工具可以添加部署到本机的工程，并显示工程中存在的各Ioserver应用、服务端应用、客户端应用，分辨出其是否正在运行。对于未运行的应用，可以调用对应的运行器启动其运行；对于已在运行的应用，可以使其对应的运行器停止（退出）运行。

此外，应用运行管理工具还可以显示本机所有正在运行的应用。

## 26.2启动应用运行管理工具

在系统的开始菜单的SCADA目录下找到“应用运行管理工具”单击启动，启动后界面如下图所示。

启动“应用运行管理工具”时，启动界面信息获得的规则为：

1.        
 “部署到本机的工程”列表中的信息是从SCADA安装目录的bin目录下的RunnerManager.dat（安装工程安装包时，会向该文件中添加安装的工程的信息）中获得的。

2.        
如果RunnerManager.dat被更名或删除，“部署到本机的工程”或“选中的工程中部署到本机的应用”列表为空白。

3.        
“本机中所有正在运行的应用”列表中的信息是通过检测当前时间本机中有哪些应用正在运行而获得的（通过进程ID）。

图 26- 1 
应用运行管理界面

 “部署到本机的工程”列表中显示了工程的信息，包括： 工程名称、简要描述、路径。

“选中的工程中部署到本机的应用”列表中显示了应用的信息，依次为： 应用类型、应用名称、描述、路径、运行状态、创建人、创建时间、修改时间。其中，运行状态有四种情况：Stopped，Starting，Running，Stopping。

应用的Starting和Stopping这两种运行状态是由该工具执行的“运行”和“停止”命令决定的。

执行“运行”命令，则应用的运行状态由“Stopped”变为“Starting”：

成功启动运行后，运行状态由“Starting”变为“Running”；

若启动运行失败但存在运行该应用的进程，运行状态保持“Starting”不变，此时用户需要到运行系统弹出的提示对话框中查看未启动该应用的原因；

若启动运行失败并且不存在运行该应用的进程，运行状态由“Starting”变为“Stopped”。

执行“停止”命令，则应用的运行状态由“Running”变为“Stopping”：

成功停止运行后，运行状态由“Stopping”变为“Stopped”；

若停止运行失败并且存在运行该应用的进程，运行状态保持“Stopping”不变，此时用户需要到运行系统弹出的提示对话框中查看未停止该应用的原因；

若停止运行失败并且不存在运行该应用的进程，运行状态由“Stopping”变为“Stopped”。

“本机中所有正在运行的应用”列表中列举了本机所有正在运行的应用。信息依次为：应用类型、应用名称、描述、路径、运行状态、CPU、内存使用和虚拟内存大小。其中，“运行状态”有两种情况：Running，Stopping；“CPU”字段显示的数值为当前运行的应用所使用的CPU占本机中所有进程使用的CPU的百分比。

运行“应用运行管理工具”过程时，界面信息显示的规则为：

1.        
如果RunnerManager.dat被更名或删除，不对“部署到本机的工程”中的信息做实时更新，只有再次打开该工具时才去读RunnerManager.dat文件。

2.        
当工程被更名或删除时，不对“部署到本机的工程”中的信息做实时更新，只有再次打开该工具时才去读RunnerManager.dat文件。被更名或删除的工程将不显示，但不会给提示。

3.        
选中工程之后，才会去读应用的信息。

4.        
如果应用读取失败，会弹出提示对话框，加载失败的应用前面会打上无效标示。

5.        
“本机中所有正在运行的应用”列表中信息会进行实时刷新，刷新频率为1.5秒。

## 26.3按钮和右键菜单

主界面的按钮包括：添加工程、移除工程、运行和停止。

**添加工程：**将本机中已经存在的工程添加到“部署到本机的工程”列表中。

**移除工程：**将选中的一个或多个工程从“部署到本机的工程”列表中移除。当未选择工程时，该按钮灰显不可用。

**运行：**运行在“选中的工程中部署到本机的应用”列表中选中的一个或多个未运行的应用，没选中任何应用、选中的应用都是无效应用时或选中的应用都是Running状态时，该按钮显示灰色不可用。

**停止：**停止在“选中的工程中部署到本机的应用”列表中选中的一个或多个正在运行的应用，没选中任何应用、选中的应用都是无效应用时或选中的应用都是Stopped状态时，该按钮显示灰色不可用。

## 26.5 启动一个或多个未运行的应用

选择图31-1“部署到本机的工程”列表中的某个工程，在“选中的工程中部署到本机的应用”列表中将会显示出该工程中部署到本机的应用。应用的运行状态是通过该工具的检测而获得的。

在“选中的工程中部署到本机的应用”列表中选中一个或多个未运行的应用，单击“运行”按钮或右键菜单的“运行”选项，即可启动运行选中的应用。

启动未运行的应用的规则为：

1.        
按照先 “服务端应用”后“客户端应用”的顺序逐一启动选中的各应用（同种类型的应用按在列表中排列的顺序由上到下启动运行）。

2.        
成功运行的应用的“运行状态”由Stopped变为Running。

3.        
当本机中已存在一个服务端应用或客户端应用正在运行（这里，正在运行的应用可以是用户手动运行的，也可以是用该工具启动运行的）时，再单击“运行”命令启动服务端应用和客户端应用，会弹出如下图所示的提示对话框。

## 26.6 停止一个或多个正在运行的应用

在“选中的工程中部署到本机的应用”中选择一个或多个应用，单击“停止”按钮或选择右键菜单的“停止”选项，完成停止运行应用的功能。

在“本机中所有正在运行的应用”列表中选择一个或多个应用，选择右键菜单的“停止”选项，即可停止正在运行的应用。

停止正在运行的应用的规则为：

1.        
按照先“客户端应用”、再“服务端应用”的顺序逐一停止运行选中的各应用（同种类型的应用按排列的顺序由下到上停止运行）。

2.        
在停止运行的过程中显示的“运行状态”为Stopping，成功停止运行的应用的“运行状态”为stopped。

3.        
“运行状态”为Stopping 的应用会在“本机中所有正在运行的应用”列表中显示。
