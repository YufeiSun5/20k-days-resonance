## 16.1     概述

在制造领域，配方是用来描述生产一件产品所用的不同配料之间的比例关系。配方是生产过程中一些变量对应的参数设定值的集合。例如，一个面包厂生产面包时有一个基本的配料配方，此配方列出所有要用来生产面包的配料成份表（如水，面粉，糖，鸡蛋，香油等）。另外，也列出所有可选配料成份表（如果酱，维生素，巧克力等），而这些可选配料成份可以被添加到基本配方中用以生产各种各样的面包。下表为某一面包厂生产面包时的配方：

|  |  |  |  |
| --- | --- | --- | --- |
|  | 配方1 | 配方2 | 配方3 |
| 配料名 | 果酱面包 | 巧克力面包 | 维生素面包 |
| 水 | 200克 | 200克 | 200克 |
| 面粉 | 4500克 | 4500克 | 4500克 |
| 盐 | 325克 | 325克 | 325克 |
| 糖 | 500克 | 500克 | 500克 |
| 鸡蛋 | 10个 | 10个 | 10个 |
| 香油 | 300克 | 300克 | 300克 |
| 水果 | 5个 | 0 | 0 |
| 巧克力 | 0 | 500克 | 0 |

    在实际生产时可根据不同的需要生产各种不同口味的面包。KingSCADA提供的配方管理器可以方便地实现配方的建立、删除、编辑等功能。

## 16.2     KingSCADA中的配方管理

KingSCADA的配方管理由两部分组成：配方模板编辑器和配方函数集。配方编辑器用于创建和维护配方模板文件；配方函数用来在KingSCADA运行时对配方模板文件中的各种配方进行选择、修改、创建和删除等操作。

所有配方都是在配方模板文件中定义和存储的，每一个配方模板文件以扩展名为csv 的文件格式存储，一个配方模板文件是通过配方定义模板产生的。

### 16.4.1    创建配方模板

配方模板创建过程如下：

**第一步**：在KingSCADA工程设计器的树型目录中选择“脚本”---“配方模板”选项→在右侧显示区工具栏中单击“新建”按钮弹出配方模板编辑器（以编辑好的模板为例），如图16-1所示：

图16- 1 配方模板编辑器

**第二步：设置配方项**

**名称：**设置配方模板的名称，限制32个字符。

**描述：**对配方模板的说明，限制32个字符。

**配料名区域：**在配料名区域中输入配方成份，如水、面粉、盐、糖等，限制16个字符。成份种类数量：1~999。

**变量集区域：**在变量集区域中输入配方成份所对应的KingSCADA变量，如水对应KingSCADA变量Water，面粉对应KingSCADA变量Flour，盐对应KingSCADA变量Salt等。一组配方成份可以对应多组变量集，如水成份除了对应Water变量外，还可以对应Water1、Water2等，最多16组。

**变量集名区域：**在变量集名区域中输入变量集的名称，限制16个字符。

**变量类型区域：**在变量类型区域中选择KingSCADA变量对应的数据类型，可选择离散型、整型、浮点型和字符串型。

**配方名区域：**在配方名区域中输入配方的名称，限制16个字符。如果酱面包、巧克力面包等。配方数最多256组。

**配方值区域：**在配方值区域中直接输入各项配方所需的数据。

**优先级：**设置配方模板的操作权限。

**安全区：**设置配方模板的安全区域。

**行/****列设置：**单击此按钮，弹出对话框，如图16-2所示：

图16- 2 配方模板行/列设置对话框

在此对话框中设置配方模板行或列，包括配料项数量、变量集数量和配方数量，也可以通过“添加项”、“删除项”按钮添加或删除配料项数量；通过“添加单元”、“删除单元”按钮添加或删除数据集数量；通过“添加配方”、“删除配方”按钮添加或删除配方数量。

对于配方模板的任意一行或任意一列可通过“行上移”、“行下移”、“列左移”、“列右移”按钮实现行/列位置的调换。

***注：在图16-1******配方模板编辑器中，当光标处于某个单元格中时按下回车键可以跳转到下一个单元格，按下Esc键，取消单元格中当前输入内容。***

**导入：**此命令可将导出的配方模板文件导入到当前配方模板中。单击此按钮在弹出的文件浏览器中选择要导入的配方模板文件的路径及文件名，即可将已存在的配方模板文件导入到当前配方模板中。

**导出：**此命令可将当前配方模板导出为.csv格式的文档，单击此按钮在弹出的文件浏览器中选择配方模板保存的路径及文件名即可。

**第三步：**配方模板编辑完成后单击“确定”按钮，系统会自动对配方进行容错检查，包括配方模板中的变量是否存在、配方模板中的变量类型是否与设置的数据类型一致等，如果检查有问题，系统会弹出相关错误信息，直到修改正确后方可将配方模板保存在当前工程路径下的Recipe目录下（.csv格式的文档），同时配方模板自动添加到工程开发环境中，如图16-3所示：

图16- 3 编辑完成的配方模板

选中并双击编辑完的配方模板可重新打开模板，对其进行重新编辑。
