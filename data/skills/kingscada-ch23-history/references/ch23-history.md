# 第二十三章 历史记录

 

 

数据存储功能对于任何一个工业系统来说都是至关重要的，随着工业自动化程度的普及和提高，工业现场对重要数据的存储和访问的要求也越来越高，为了满足数据存储和访问的要求，KingSCADA提供了下面两种数据存储方式：

方式一：将数据存储到KingSCADA自身提供的高速历史数据库（后面统称为KingSCADA历史库）。

方式二：将数据存储到工业库KingHistorian。

第二十三章 历史记录

## 23.1         数据存储到KingSCADA历史库

KingSCADA提供了高速历史数据库，支持毫秒级高速历史数据的存储和查询，采用最新数据压缩和搜索技术，大大节省了磁盘空间，查询速度也有了明显的提高。同时为了保证历史数据和报警信息能够完整地保存到历史数据库和报警数据库中，KingSCADA提供了数据缓存模式，真正地解决了数据丢失的问题。

注意：因停电或迁移工程环境，造成时间错误，一定要要先调整时间，然后再启动SCADA。否则会造成历史数据异常，例如：归档失败，归档数据丢失等情况。严重的会造成所有数据时间戳错乱，数据不可恢复。

### 23.1.1   KingSCADA变量的历史记录属性

在KingSCADA中，离散型、整型和实型变量支持历史记录，字符串型不支持历史记录，历史记录属性设置如下：

在KingSCADA工程设计器的数据词典中双击某一变量，在弹出的变量属性对话框中选择“历史”选项卡，如图23-1所示：

图23-1变量的历史记录属性设置

记录属性的定义：

**无：**选择该项，则该变量不进行历史记录

**改变：**系统运行时，变量的实时值在发生变化，如果变量的当前值与上一次记录的历史数据值之间的差值大于设置的死区范围时，该变量的值才会被记录到历史库中，这种方式适用于数据变化较快的场合。

**间隔：**无论变量变化与否，系统运行时按定义的时间间隔将变量的值记录到历史库中，每隔设定的时间对变量的值进行一次记录。最小定义时间间隔单位为1秒，范围1-3600。这种方式适用于数据变化缓慢的场合，缺省设置为60秒。

**备份：**有些设备带有一定数量的数据存储器，可以存储一段时间的设备采集到的数据，但这些设备往往只是简单的记录数据，而不能进行历史数据的查询、浏览等操作，而且必须通过上位机的处理才可以看到。此时选择该项，KingSCADA在运行时，不再直接向历史库中记录该变量的数值，而是通过驱动程序将设备中存储的历史记录读取上来，并按照约定的时间格式和变量类型插入到KingSCADA的历史库中，这样保证了历史库的完整性。

**每次采集记录：**系统运行时，按照变量的采集频率进行数据记录，每到一次采集频率，记录一次数据。该功能只适用于IO变量，内存变量没有该记录方式。该功能应慎用，因为当数据量比较大，且采集频率比较快时，使用“每次采集记录”，存储的历史数据文件会消耗很多的磁盘空间。

下面介绍三种方式间接查看KingSCADA里面哪些变量设置了历史数据：

1)       
添加一个历史趋势曲线，在曲线的dataSource里关联变量时可以看到枚举出来的设置了历史存储的变量。

2)       
添加一个报表，在报表属性的HistoryTags里关联变量时可以看到枚举出来的设置了历史存储的变量。

3)       
将数据词典中的变量导出，查看字段HisRecMode，数值为1即设置了历史保存，但这种方式不适用与数据模型。

### 23.1.2   历史数据存储及文件格式

KingSCADA的历史数据被存储到历史库中，历史库的存储路径及历史数据的存储天数可以通过下面方式进行设置：

在KingSCADA工程设计器的树型目录中双击“系统设置”→“历史库服务设置”选项，弹出历史库服务设置对话框，选中“使用历史库”选项，如图23-2所示。

图23-2历史库服务设置对话框

**数据保留时间（天）：**输入历史数据保存的时间，最长为8000天，最短为1天，当到达设置的时间时，系统自动删除最早的历史记录文件。历史记录文件有下面四种类型的文件：

**（1****）临时数据文件（\*.ArcTmp****）**

存储最近一段时间的原始数据。

文件名格式为YYYYMMDDHHXXXXXXX.AcrTmp，YYYY—年，MM——月，DD——日，HH—小时，XXXXXXX——文件序号。年、月、日、小时的数值是根据生成数据时刻的格林威治时间确定。若在临时文件没有被压缩到arc文件前，重新启动，会重新生成一个临时文件

**（2****）归档文件（\*.arc****）**

保存的是一天内的压缩数据，文件名格式为YYYYMMDDXXXXXXX.arc

**（3****）特征值文件（\*.ev****）**

存放的是原始数据特征值，文件名格式为YYYYXXXXXXX.ev

**（4****）临时特征值文件（\*.EvTmp****）**

这是存储临时特征值数据的文件，将根据特征值算法过滤出的数据，在下个月到来归档之前，先存放在临时特征值文件中。文件名格式为YYYYXXXXXXX.EvTmp。

***注意：KingSCADA******历史库文件用户是不能够修改，也不能用其它应用程序打开，只能够进行查询。***

***KingSCADA******历史库文件是以文件格式保存在计算机里的，可以删除掉的，但是要注意如果一旦删除此文件，将无法查询到该时期的历史数据。如果不慎将历史库文件删除的话，将无法进行恢复***

**最小剩余磁盘空间（兆）：**磁盘存储空间不足时报警。当历史库文件所在的磁盘空间小于设置值时（设置范围100-8000），系统运行后将检测存储路径所在的硬盘空间，如果硬盘空间小于设定值时，系统将会在信息窗口给出报警提示，此时工程人员应该尽快清理磁盘空间，以保证KingSCADA历史数据能够正常保存。

**历史数据存储路径：**历史数据存储路径可以选择默认路径（即当前工程路径）或者指定某个路径。如果选择当前工程路径的话，历史数据保存到当前工程路径下的HistoryData目录中，如果指定路径的话，历史数据保存到指定的目录下。

### 23.1.3   缓存区设置

为了保证历史数据和报警信息能够完整地保存到历史数据库和报警数据库中，KingSCADA提供了缓存区设置的功能。

如果选择了缓存模式，那么当KingSCADA产生新的历史数据或报警信息时，新数据会通过缓存文件发送给历史数据库或报警数据库，如果数据库收到数据，缓存会清掉发送成功的数据。如果数据库服务器有问题，收不到数据，那么发送失败的数据会保存在缓存文件里，当数据库服务器修复后，系统会把缓存文件的数据发送给它们，发送成功的数据就被删掉。如果缓存的数据量很大，要很长时间才能发送完，在把缓存数据发送完之前的这段时间里，客户端是无法查询到最新的历史数据和报警信息的。

为了避免过于频繁的写磁盘缓存文件操作，新数据首先进内存缓存，内存缓存满了，再写到磁盘里去。内存缓存大小用户是可以设置的，如果设置太小，会造成硬盘的频繁读写，如果设置过大，在有故障情况下，可能丢失的最新历史数据较多，建议用默认值8M。

磁盘缓存区能够缓存的最大数据量是由磁盘剩余空间决定，用户可以设置单个缓存文件的大小，当缓存数据超过设置的缓存文件大小时，就会再生成一个新的缓存文件。

如果没有选择缓存模式，那么当KingSCADA产生新的历史数据或报警信息时，新数据会被直接发送给历史数据库和报警数据库，不通过缓存，如果此时数据库有问题，收不到数据，发送失败的数据无法保存在缓存文件里，数据就会丢失。

缓存区设置方法如下：

在KingSCADA工程设计器的树型目录中双击“系统设置”→“高级设置”选项，弹出对话框，如图所示：

图23- 3高级设置对话框

**历史数据通过数据缓存区：**选中该项，表示历史数据存储选择了缓存模式，并设置内存缓存区和单个缓存文件的大小。

**报警数据通过数据缓存区：**选中该项，表示报警信息存储选择了缓存模式，并设置内存缓存区和单个缓存文件的大小。

**优化数据采集：**此项参数可以优化KingSCADA的数据采集。如果选中“优化数据采集”选项的话，则以下任一条件满足时KingSCADA将执行数据采集任务（从IOServer中采集数据）：A、当前显示画面上正在使用的变量；B、历史数据库正在使用的变量；C、报警记录正在使用的变量；D、命令语言中正在使用的变量。任一条件都不满足时将不采集；当不选择“优化数据采集”选项时，KingSCADA将按变量的采集频率周期性地执行数据采集任务。

## 23.4         画面历史回放

画面历史回放功能，指在运行态中通过调用历史回放函数，实现某个画面的历史回放，即画面上的动画连接不显示当前时刻的状态，而是显示之前的指定时刻或时间段的状态。例如在画面上建立了某变量的输出动画，在画面进行历史回放时，该变量输出动画显示的是指定时刻或时间段的变量值。

 画面历史回放类型共计两种，分别是历史时刻回放和历史时段回放，历史时刻回放显示的是指定历史时刻的历史数据，历史时段回放显示的是过去某个时段内的历史数据。

**1****、int KGSetPictureMode(string strPicName,

用户可以通过本脚本函数来切换画面的状态，实现历史回放功能以及恢复正常显示的功能。任何时候都可以调用本函数进行画面运行模式切换（除了上一个命令未执行完之外）。

历史回放有两种类型：1、历史时刻回放； 2、历史时间段回放。

用户可以通过本脚本函数来确定是采用哪种回放类型。只有当画面处于历史回放模式时，本脚本函数才起作用。当正在回放时，不允许修改回放类型。

**参数：**

strPicName：画面名称字符串，不能是该函数所在画面的名称；

iReplayType：设定的历史回放类型。取值：0 历史时刻回放；1 历史时段回放。

**3****、int KGSetSamplingMode(string
strPicName, int iSamplingMode)**

**功能：**

设置画面历史回放时生成数据的采样模式。

数据的生成有两种模式：1、线性插值采样模式； 2、原始值采样模式。用户可以通过本脚本函数来确定是采用哪种数据采样模式。

无论数据库中的数据是压缩存储还是非压缩存储，数据库中的数据点都是不连续的，某一时刻不一定存在对应的数据点；当设定时刻的数据点不存在时，就需要通过某种方式得到，由此引出采用模式－－数据点生成的方式。

线性插值采样模式，是指历史回放数据的生成方式是采用线性插值方式得到的，具体的说，某一时刻的数据值是通过前面一个数据点和后面一个数据点进行数据值的线性插值得到。原始值采样模式，实际上是梯形采样模式，是指某一时刻的数据点如果没有数据库存储值，则直接采用数据库中前一个存储点的值。对于KingGraphics 和 KingSCADA等界面程序来说，只有变化的数据点才进行处理，所以梯形采样模式实际表现就是原始值采样模式，历史回放的是数据库中存在的实际存储值。

只有当画面处于历史回放模式时，本脚本函数才起作用。当正在回放时，不允许修改采样模式。

**参数：**

strPicName：画面名称字符串，不能是该函数所在画面的名称；

iSamplingMode：设定的采样模式。取值：0 线性插值采样；1 原始值采样。

**举例：**

int  KGSetSamplingMode(“PIC1”,1);

将画面PIC1的数据采样模式设置为原始值数据采样。

**返回值：**

0：成功；1：执行错误；2：忙（上次命令执行中）；3：未处于历史回放状态；4：回放状态不允许（正在回放）；5：输入参数越限；6：输入参数格式错误；7：未找到该画面。

**4****、****int KGSetHisReplaySpeed(string strPicName, float fSpeed)**

**功能：**

设置画面的历史回放速度。

回放速度只对历史时间段回放有效，是指在该时间段内回放时的时间流速是正常时间流速的倍数，即通常意义上的“快放”、“慢放”。用户可以通过本脚本函数来确定回放的速度。只有当画面处于历史回放模式时，本脚本函数才起作用。任何时候都可以调用本接口调整回放速度（除了上一个命令未执行完之外）。

**举例：**

int  KGSetHisReplaySpeed(“PIC1”, 2.0);

设置画面PIC1的回放速度为原始速度的2.0倍

**返回值：**

0：成功；1：执行错误；2：忙（上次命令执行中）；3：未处于历史回放状态； 5：输入参数越限；6：输入参数格式错误；7：未找到该画面。

**注意：回放速度可设置为****0.5****的倍数，例如****2.0,5.0****，不支持****2.1****或者****5.3****这类的回放倍数。**

**5****、int KGSetHisReplayPeriod(string

#### 23.4.2.2   操作画面历史回放函数

**1****、int KGStartHisReplay(string
strPicName)**

**功能：**

开始画面历史回放。本函数只有在画面处于历史回放运行模式时进行历史时间段回放才有效，用户可以通过本脚本函数来控制回放的开始。

**参数：**

strPicName：画面名称字符串，不能是该函数所在画面的名称。

**举例：**

int  KGStartHisReplay(“PIC1”);

设置画面PIC1开始进行画面历史回放

**返回值：**

0：成功；1：执行错误；2：忙（上次命令执行中）；3：未处于历史回放状态；4：回放状态不允许（正在回放）；6：输入参数格式错误；7：未找到该画面。

**2****、int KGStartHisReplayEx(string
strPicName, int iReplayType, int iSamplingMode, float fSpeed, string
strStartTime, string strEndTime)**

**功能：**

扩展的开始画面历史回放脚本函数。本扩展函数同时设定历史回放的参数和启动历史回放。

本函数只有在画面处于历史回放运行模式时才有效，用户可以通过本脚本函数来控制回放的开始。

对历史时间段回放，本扩展脚本函数和 KGStartHisReplay() 一样采用异步方式实现的；而对于历史时刻回放，马上执行。历史时间段回放的最长时间为1天，当超过一天时，自动调整终止时间到一天上。

**参数：**

strPicName：画面名称字符串，不能是该函数所在画面的名称；

iReplayType：设定的历史回放类型。取值：0 历史时刻回放；1 历史时段回放；

iSamplingMode：设定的采样模式。取值：0 线性插值采样；1 原始值；

fSpeed：设定的历史回放速度。指实际速度的倍数，例如 1.0 表示原速度；

strStartTime：开始时间字符串。格式：YYYY-MM-DD HH:mm:ss；

strEndTime：结束时间字符串。格式：YYYY-MM-DD HH:mm:ss。

**举例：**

int  KGStartHisReplayEx(“PIC1”, 1, 1, 2.5, “2021-8-10
14:00:30”, “2021-8-11 12:00:30” );

设置画面PIC1按照历史时段从2021-8-10 14:00:30——2021-8-11 12:00:30这段时间进行2.5倍速的历史回放，采样模式选用：原始值采样。

**返回值：**

0：成功；1：执行错误；2：忙（上次命令执行中）；3：未处于历史回放状态；4：回放状态不允许（正在回放）；5：输入参数越限；6：输入参数格式错误；7：未找到该画面。

**3****、int KGPauseHisReplay(string

### 23.4.4   画面历史回放需知

1)       
该功能目前只支持本地内存变量、IO变量、数据模型实例变量的历史回放功能，不支持引用变量和远程站点变量的历史回放。

2)       
画面进入历史回放状态后，该画面的所有画面脚本（基本脚本、定时脚本、变量改变脚本）和触发式脚本（需要点击或滑动鼠标来触发的脚本）都不再执行。

3)       
只有设置了工业库历史存储的变量才能够进行历史回放。

4)       
画面从实时模式切换到回放模式后，启动回放之前，画面的动画连接数据保持为切换前的数据。

5)       
历史回放结束后状态：停止回放，不会重复循环回放，也不会自动回到实时状态。

6)       
画面的历史回放和实时状态切换改变都是通过设置画面运行模式函数来实现。

7)       
实时曲线、历史曲线、历史报表、报警窗口、控件不能进行历史回放。

8)       
历史回放时，实时曲线不回放，也不显示当前的实时曲线，实时曲线停止绘制。历史曲线是根据查询的时间段来显示曲线的，历史回放时，历史曲线不回放，也不能对历史曲线进行操作，因为回放时脚本等都停止运行了。

9)       
只能操作其他画面进行历史回放，不能操作本画面回放。

10)    
历史回放时，关闭画面后，重新打开该画面，画面就恢复成实时状态了。历史回放时，打开了其他画面，切换到其他画面去时，该画面还一直在进行回放。

11)    
切换画面运行模式的脚本函数(KGSetPictureMode)不能放到系统脚本和事件脚本里。

12)    
不支持客户端和web端进行历史回放
