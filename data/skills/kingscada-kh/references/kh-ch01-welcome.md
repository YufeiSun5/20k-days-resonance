# 第1章     欢迎使用KingHistorian

KingHistorian管理工具是对KingHistorian工业库进行管理和配置的可视化用户接口，可以帮助用户实现数据监控、存储、查询等操作，及根据需求对数据库和客户端的程序进行配置管理，操作简单，功能强大。

KingHistorian系统工具包括四个组件，即KingHistorian，管理工具，OPC采集器，和二次开发接口(API，OLE DB Provider，COM自动化接口和JavaAPI接口)。

在运行安装程序之前，请先定义好整个系统的结构，决定在每个结点计算机上安装哪些组件。如果想把所有的组件都安装在同一台机器上，只要在这台机器上运行安装程序即可。如果计划使用多台机器作为服务器、数据采集器，或客户端，请分别在各个机器上运行安装程序以安装需要的组件。

## 1.1 KingHistorian系统组件

KingHistorian系统包括四个组件，即KingHistorian，管理工具，OPC采集器，IOServer采集器，和二次开发接口(API，OLE DB Provider，COM自动化接口和JavaAPI接口)。

l      
KingHistorian

KingHistorian数据服务器是KingHistorian系统的核心，主要实现功能包括：

（1） 检索变量数据：根据用户输入的查询条件，检索符合条件的变量历史数据和变量实时数据，并将结果返回给客户端。

（2） 处理变量数据：插入、修改、删除、压缩和存储变量历史数据。

（3） 变量管理：变量（标签点）的配置和管理（组态），包括一般的变量和计算变量。

（4） 数据采集器管理：监控连接到数据库系统的数据采集器状态，管理和控制数据采集器的工作。

（5） 结构化数据存取：允许使用类似于关系数据表的方式来存取结构化数据，结构化存储方式可以用来存储事件、用户角色、变量配置和系统配置数据。

（6） 安全管理：与数据服务器安全性相关的用户、角色、权限管理，包括用户的新建、修改、删除和授权，角色的新建、修改、删除和授权等。

（7） 系统管理和监控：系统的管理和运行监控，变量管理、数据采集器管理、用户和权限管理都属于系统管理的一部分，除此之外还包括数据表、数据文件、日志的管理，以及数据的备份、还原等。

l      
管理工具

KingHistorian系统管理工具为KingHistorian提供可视化的管理和配置接口。它可以根据用户的角色权限配置、修改、使用不同的功能组件，实现采集器配置和管理、变量配置和管理、用户管理、SQL语句查询等功能。

l      
OPC采集器

OPC数据采集器架构在OPC规范之上，可以对OPC服务器进行实时数据采集。所有工业现场采用OPC服务器方式的，均可使用“OPC数据采集器”采集数据。

l      
二次开发接口(API，OLE DB Provider，COM和JavaAPI接口)

为了用户更方便地使用KingHistorian，我们提供了一套可对KingHistorian数据服务器进行连接和操作的函数接口，即SDK开发包，用户可以在编写自己的程序中调用这些函数，通过对参数进行简单的赋值，就可以实现服务器连接，变量配置，采集器配置，用户和安全配置等复杂的操作。

KingHistorian的SDK开发包由四部分组成，API接口、OLE DB Provider、COM自动化接口和JavaAPI接口。用户可以根据实际情况选择这些接口开发自己的客户端程序。

### 1.2.2  软件要求

## 1.4 卸载KingHistorian系统程序

卸载KingHistorian系统工具有两个方法：

       **（一）运行KingHistorian****安装包中的Setup.exe****安装程序来卸载**

在光盘驱动器中插入“KingHistorian”软件的安装盘，系统自动启动setup.exe安装程序，弹出如下图所示的对话框：

图1-
15 准备安装对话框

图1-
16 欢迎使用KingHistorian 安装向导

图1-
17 选择删除程序

选择“删除”选项，单击“下一步”，则弹出确认删除程序的对话框，如下图所示：

图1-
18 确认删除程序

单击“删除”按钮，开始卸载KingHistorian程序，

图1-
19 开始卸载KingHistorian程序

图1-

在对话框中找到KingHistorian，然后单击“删除”按钮，即可卸载程序。
