第五章
控件

## 5.1      概述

控件是可重用对象，用来执行专门的任务。每个控件实质上都是一个微型程序，但不是一个独立的应用程序，通过控件的属性、方法等控制控件的外观和行为，接收输入并提供输出。

控件在外观上类似于组合图素，工程人员只需把它放在画面上，然后配置控件的属性，进行相应的函数连接，控件就能完成复杂的功能。另外，控件的可重用性也给工程开发人员提供了方便，极大地提高了工程人员工程开发和工程运行的效率。

## 5.2      支持的Windows UI控件介绍

KingSCADA把Windows控件封装成图形系统的基本图素，当需要使用时，直接从工具箱中拖拽出即可，Windows控件包括：按钮、编辑框、复选框、组合框、列表框、单选按钮、日历控件、复杂文本框、树型控件、日期控件、相对时间控件。如下图所示：

图5-1 windows UI控件

控件所具有的公共属性如下：

**Name****：**控件对象的标识，长度不超过40个字符。该属性在属性框中以文本方式显示，当对象创建时可由系统默认命名，如UIButton1、UIButton2….可进行手动修改。

**MemberAccess****：**设置该控件是否能在脚本中访问，True：能在脚本中访问，False：不能在脚本中访问，默认是True。

**Comment****：**控件的描述信息长度不超过128个字符；该属性在属性框中以文本方式显示，当对象创建时默认为空，可进行手动修改。

**Location****：**控件在画面上的位置信息，以像素为单位

**Size****：**控件的长度和宽度，以像素为单位

**EditLock****：**设置该控件在开发环境中是否被锁定

**EditVisible****：**设置该控件在开发环境中是否显示

**SecurityPrivate****：**设置控件操作的优先级别。

**SecuritySection****：**设置控件的安全区域

**Visible****：**设置该控件在运行环境中是否显示

**Enable****：**设置该控件在运行环境中是否可以进行操作

**TabIndex****：**画面添加UI控件（包括文本框）后，其属性中增加了TabIndex属性，如图中所示，TabIndex默认值为0，该整数值可修改为其他整数值，代表运行界面上Tab键切换时焦点选择各控件的顺序。

### 5.2.2    文本框

文本框是可以在用户界面中进行各种数据输入的控件。

**1、** **文本框控件特有属性介绍**

在KingSCADA画面编辑器中打开任一画面，执行"对象"菜单 →"UI 控件"→"文本框"命令或单击Windows UI工具箱中的图标，此时把鼠标放在画面编辑区，鼠标将变成小"十"字形，在画面上需要插入控件的地方按下鼠标左键，拖动鼠标，画面上出现一个矩形框，表示创建后控件界面的大小。松开鼠标左键，控件在画面上显示出来。其控件属性对话框，如下图所示：

图5-5 文本框属性对话框

**BorderStyle****：**设置文本框显示方式

**BackgroundColor****：**设置文本框内部背景色

**Font****：**设置文本字体及大小

**FontColor:** 设置文本的字体颜色

**Text****：**输入文本框显示的文本

**Layout****：**当文本内容为多行且MultiLine属性为True时，此属性用来设置文本对齐方式，有左对齐、右对齐和中心对齐三种方式

**MultiLine****：**当输入的文本内容超过编辑框的长度时，文本是否多行显示

**PassWord****：**是否以"\*"字符显示输入的文本。当文本框中输入密码时，可以选择True

**ReadOnly****：**设置文本框的只读属性，当选择True时，则运行时不允许进行输入操作

**MaxLength****：**设置文本框输入文本的最大字符长度，范围：0-2147483647

**ScrollBar****：**设置当文本内容超出文本框的长度时是否显示滚动条，有无滚动条、横向滚动条、纵向滚动条和横向纵向滚动条三种选项，该项仅在MultiLine属性设置为True时有效，否则只能选择None

**3****、文本框控件属性赋值**

当文本框控件的MultiLine属性为True时，在脚本中给文本框的Text属性赋值时，如果希望文本框显示多行文本，需要在文本字符串换行处插入回车和换行符的ASCII码。例句：

添加列表项

方法名称：

Int AddItem(String ItemText);

方法功能：

向组合框控件中添加列表项。KingSCADA将增加的列表项作为列表框中的一个成员项——Item，并自动给这个成员定义一个索引号，索引号从0开始逐渐增加

参数说明：

ItemText：要添加的列表项的内容，可以是字符串变量，也可以是字符串常量，如"abc",则执行此方法将"abc"添加到控件内容列表中；如为字符串变量则将字符串变量当前值添加到内容列表中

返回值：

整型值：返回添加项的索引号。

调用格式：

UIComboBox1.AddItem("Text1");

插入列表项

方法名称

Bool InsertItem(Int nIndex,String
ItemText);

方法功能：

向组合框控件中的指定位置插入列表项

（4）

True：删除成功

### 5.3.1    OCX控件概述

对象链接和嵌入用户控件(OCX）是一种可以由在微软的Windows系统中运行的应用软件创建使用的特殊用途的程序。

KingSCADA支持Windows OCX控件，包括Microsoft提供的标准ＯＣＸ控件和用户自制的控件。ＯＣＸ控件的引入在很大程度上方便了用户，用户可以灵活地编制一个符合自身需要的控件，或调用一个已有的标准控件，来完成一项复杂的任务，而无须在KingSCADA中做大量复杂的工作。一般的控件都具有控件属性、控件方法、控件事件，用户在KingSCADA中通过调用控件的这些属性、事件、方法来完成工作。

strDatasetName：数据集名称，字符串类型。在设置控件的数据集之前，应事先通过KDBGetDataset函数建立数据集。

（4）查询并显示指定变量的工业库中的历史记录

int QueryandShowTagsData(int iTableMode,string ConnectStr,int
iTagNamesType,string strTagNames,string strStartTime,string strEndTime,int
iSamplingMode,int iSamplingInterval,int iThreadCount);

参数：

iTableMode：展示历史记录的形式，0为窄表显示，大于0为宽表显示。

ConnectStr：工业库连接字符串，连接字符串ConnectStr ="
ServerAddress=127.0.0.1;ServerPort=5678;UID=sa;pwd=sa;NetworkTimeout=0"，其中超时单位为毫秒，0表示不超时退出。

iTagNamesType：strTagNames格式类型，1=变量名称组，2=csv文件路径。

strTagNames：工业库变量名称组和或者csv文件路径。

strStartTime：起始时间字符串。规定格式：YYYY-MM-DD HH:mm:ss。

strEndTime：终止时间字符串。规定格式：YYYY-MM-DD HH:mm:ss。

iSamplingMode：采样模式。0 原始值采样（宽表不支持原始值采样）;1 线性插值采样;2 步进插值采样。

iSamplingInterval：采样间隔，单位：毫秒。

iThreadCount：线程个数，1~12。

 

返回值：

0：执行成功。

非0：执行失败。

错误码：

-1：参数错误，例如参数小于0，参数为空。

-2：解析变量名称出错或变量名为空（csv文件中的变量名为空）。

-4：解析服务器地址、端口号、用户名、密码等错误

-5：查询工业库失败

## 5.5      历史趋势曲线控件

KSHTrend曲线控件是KingSCADA以OCX控件形式提供的绘制历史曲线和ODBC数据库曲线的功能性工具。该曲线具有以下特点：

1、 
可以连接KingSCADA的历史库，也可以连接工业库服务器，还可以通过ODBC数据源连接到其它数据库上，如Access、SQLServer等。

2、   
连接KingSCADA历史库或工业库服务器时，可以定义查询数据的时间间隔，如同在KingSCADA中使用报表查询历史数据时使用查询间隔一样。

3、   
完全兼容了KingSCADA原有历史曲线的功能。最多可同时绘制16条曲线。

4、   
可以在系统运行时动态增加、删除、隐藏曲线。还可以修改曲线属性。

5、   
曲线图表实现无级缩放。

6、   
可实现某条曲线在某个时间段上的曲线比较。

7、   
数值轴可以使用工程百分比标识，也可用曲线实际范围标识，二者之间自由切换。

8、   
可直接打印图表曲线。

9、   
可以自由选择曲线列表框中的显示内容。

10、             
可以选择移动游标时是否显示曲线数值。

11、             
可以在曲线中显示报警区域的背景色。

#### 5.5.2.1      曲线属性页

如图5-68所示，曲线标签页中右侧部分为说明定义在绘制曲线时，历史数据的来源。曲线中数据的来源，可以是KingSCADA历史库、工业库或其它关系数据库的ODBC数据源。每次添加的曲线都有默认的不同的颜色，若不满足用户需求，可自行修改。

曲线标签页中左侧部分"曲线"列表是定义曲线图表初始状态的曲线变量、绘制曲线的方式、是否进行曲线比较等：

**1****、历史库中添加**

从KingSCADA历史库中选择变量到曲线图表，并定义曲线绘制方式。单击"历史库中添加"按钮，弹出如图所示的对话框：

图5-69 添加曲线对话框

对话框中各部分的含义为：

变量名称：在"变量名称"文本框中输入要添加的变量的名称，或单击"选择变量"按钮，在弹出的变量选择器中选择。在添加、修改历史库变量对应的曲线时，可以按照变量的变量名、类型、描述、ID升序或降序排列，以方便进行变量的查找。

线类型：单击"线类型"后的下拉列表框，选择当前曲线的线型。

线颜色：单击"线颜色"后的按钮，在弹出的调色板中选择当前曲线的颜色；默认每条曲线添加的颜色是不同的。

绘制方式：曲线的绘制方式有四种：模拟、阶梯、逻辑、棒图，可以任选一种。

无效数据绘制方式：曲线在曲线变量关联的设备通讯失败，关联变量的质量戳为坏，运行系统退出的情况下显示的方式。分为：虚线，不画线，实线。

是否显示数值点数值，数据背景色：如果选中，在绘图区显示曲线各数据点的数值，同时可以设置数据点值的背景色。

隐藏曲线：控制运行时是否显示该曲线。在运行时，也可以通过曲线窗口下方的列表中的属性选择来控制显示或隐藏该曲线。

纵轴单位：可以为每一条曲线设置不同的"纵轴单位"，如果不设置图5-67中的"纵轴单位"，那么曲线的纵轴单位按照图控件"坐标系"页面中"纵轴单位"的设置显示。

小数位数：显示某变量的对应曲线时，设置该曲线数值显示的小数位数。仅当该变量是浮点型时，才起作用。不同的曲线可以设置不同的小数位数。

曲线比较：通过设置曲线显示的两个不同时间，使曲线在绘制位置有一个时间轴上的平移，这样，一个变量名代表的两条曲线中，一个是显示与时间轴相同的时间的数据，另一个作比较的曲线显示有时间差的数据(如昨天），从而达到用两条曲线来实现曲线比较的目的。

若选择从数据库中添加变量，必须先配置数据源。具体配置方法如下：

步骤一：打开"控制面板"à "管理工具"à "数据源(ODBC）"。单击"用户DSN"或"系统DSN"项，单击"添加"按钮，弹出"创建新数据源"对话框。

步骤三：在"数据源名"中定义一个数据源名称，用户可以自己设置，数据库"选择"中选择曲线要访问的数据所在的数据库，此数据库的表至少有三个字段：时间字段、数据字段、毫秒字段。单击"确定"，新创建的数据源就添加到"用户DSN"或"系统DSN"列表中。

表名称：选择曲线使用的数据来自所选数据库的某一个表。

时间字段：选择曲线数据对应的时间记录，日期/时间类型。

数据字段：选择曲线对应的数据值，离散型、长整型或浮点型。

毫秒字段：选择曲线数据对应的毫秒记录，长整型或短整型，该毫秒字段里面的数据可以不是实际的毫秒数，只要有整型数据即可。

所在时区：选择数据库中记录数据时所在的时区。需要用户定义数据来源所在的时区，缺省为当前操作系统设置的时区。如果时区选择错误的话，可能在图表的相应时间段上找不到曲线。

无效值：每一条曲线都和表中一个表示其值的字段关联，这个字段的值在某一时点可能是无效的，但表的结构决定了这个字段在一条记录中的值不能为空白，所以就有了无效值的定义。

²  当表中数值字段的值为NULL时表示该点数据无效，那么配置无效值时就可以为空。

²  当表中数值字段的值为0时表示该点数据无效，那么配置无效值时就可以写"0"。

²  当表中数值字段的值为"abcd"时表示该点数据无效，那么配置无效值时就可以写"abcd"。

② 列表项设置；

③ 列表宽度设置，列表宽度由用户在运行态拖拉列表头修改；

④ 用户在工具条上所做的修改：

⑤ 用户用脚本函数对曲线所做的修改。

***3."******在线保存"******功能的权限限制，只能和曲线控件的操作权限一起限制，通过设置曲线的SecurityPriority******、SecuritySection******属性来实现。***

#### 5.5.3.2      历史曲线控件方法列表

历史曲线控件提供了很多控件方法，供用户在命令语言中调用。下面介绍历史曲线控件方法的用法。

**1****、Void 
CallSetAttrVal(STRING strAttrName,LONG lIndex,FLOAT fAttrVal)**

功能：设置某条曲线的属性，该属性为数值类型，包括Bool、float、long、short、Double、OLE\_COLOR类型(OLE\_COLOR可以看作整数类型）。

**void CallSetAttrStrVal(STRING strAttrName,LONG
lIndex, STRING strAttrVal)**

功能：设置某条曲线的属性，该属性为字符串类型。

此两种方法只对以"Curve…"开头的有关曲线的属性有效。

参数：

strAttrName：曲线属性名称

lIndex：曲线索引号

fAttrVal：设置值，实型

strAttrVal：设置值，字符串型

例：新建历史趋势曲线控件m，定义按钮弹起时：

m.Control.CallSetAttrVal("CurveDrawType",1,0);

把第一条曲线的属性CurveDrawType设置为0，即将第一条曲线的绘制方式设为按"模拟"方式绘制。

**2****、Void
ChangeCurveVarName(LONG 1Index,STRING szNewName);**

功能：改变历史曲线所连接的变量，该变量数据来自KingSCADA历史库。

参数：

1Index：曲线索引号

szNewName：变量名

例：定义变量V1，m为历史曲线控件名称，定义按钮弹起时

m.Control.ChangeCurveVarName(1,"\\local\V1");

把第一条曲线连接的变量改成V1。

***注意：调用该方法后有时会出现数值轴为******0******的情况，但通过曲线的工具栏改变连接的变量数值轴是正常的，这种情况可以在更改曲线命令后再加入一条语句：******HT.Control.NeedRefreshCtrl=true;*** ***刷新曲线，数值轴可正确显示***

#### 5.5.4.2      工具条的使用

曲线图表的工具条是用来查看变量曲线详细情况的。工具条的具体作用可以通过将鼠标放到按钮上时弹出的提示文本看到。下面详细介绍每个按钮的作用。

**1、****隐藏****/****显示变量选择框：**单击按钮可以隐藏/显示变量选择框。运行态下显示的变量选择框如下图所示。

数据源框中列出历史库与工业库节点，选中任意节点，变量框中显示该节点下的所有变量，双击该变量，便可成功的在控件中添加该变量的曲线。

若某节点下变量数量较多，则可通过检索的方式的查询变量。检索支持通配符方式。

**2****、设置左时间：**修改控件时间轴的左时间，当时间间隔框为“自定义”时可修改，可通过“/”键切换时间的选中项。

**3****、时间间隔框：**下拉后显示可选择的时间间隔，当选择“自定义”时，工具条的左右时间可任意设定，点击刷新按钮，控件的左右时间显示为设定的时间。

当选择“昨天”表示前一天的0点到今天的0点，选择“今天”表示从今天的0点到当前时间，“上一小时”，当前时间的前一个小时取整点，“当前小时”，当前小时的整点到当前时间，例当前时间“10：14：20”，选择“上一小时”，点击刷新曲线按钮，控件的左右时间变为：“9：00：00”---“10：00：00”，选择“当前小时”，点击刷新曲线按钮，控件的左右时间变为“10：00：00”---“10：14：20”。

其他的时间间隔均是从设置的右时间的基础上减掉相应的时间间隔就是显示的控件的左右时间

**4****、设置右时间：**，修改控件时间轴的右时间，若修改的时间大于当前时间，则默认修改为当前时间，通过“/”键切换时间的选中项。

**5****、刷新曲线：**根据左右时间刷新当前控件中显示的曲线。

单击该按钮弹出如下图所示的对话框，修改当前跨度时间设定值。

图5-111 修改调整跨度

#### 5.6.4.2      工具条的使用

曲线图表的工具条是用来查看变量曲线详细情况的。工具条的具体作用可以通过将鼠标放到按钮上时弹出的提示文本看到。下面详细介绍每个按钮的作用。

**1****、调整跨度设置按钮：**

单击该按钮弹出如下图所示的对话框，修改当前跨度时间设定值。

图5-127 修改调整跨度

在"单位"列表框中选择跨度的时间单位，有：分钟、小时、日。在"跨度"编辑框中输入时间跨度的数值。

单击按钮使曲线图表向左移动一段指定的时间段。单击按钮使曲线图表向右移动一段指定的时间段。

**2****、放大按钮：**在曲线图表中选择一个矩形曲线区域，单击该按钮，或直接单击该按钮，可以放大当前的曲线图表：

²  当在曲线区域选取了矩形区域时，时间轴最左/右端调整为矩形左/右边界所在的时间，从而使曲线局部放大，左/右游标位置分别置时间轴最左/右端。

²  当未选定矩形区域时，如左/右游标不在时间轴最左/右端，时间轴最左/右端调整为左/右游标所在的时间，数值轴不变，从而使曲线局部放大，左/右游标位置分别置时间轴最左/右端。

²  当未选定矩形区域，左/右游标在时间轴最左/右端时，时间轴宽度调整为原来的一半，保持中心位置不变，数值轴不变，从而使曲线局部放大，左/右游标位置分别置时间轴最左/右端。

**3****、缩小按钮：**在曲线图表中选择一个矩形曲线区域，单击该按钮，或直接单击该按钮，可以缩小当前的曲线图表：

²  当在曲线区域选取了矩形区域时，矩形左/右边界所在的时间调整为时间轴最左/右端所在的时间，矩形上/下边界所在数值调整为数值轴最上/下端所在数值，从而使曲线局部缩小，左/右游标位置分别置时间轴最左/右端。

²  当未选定矩形区域时，如左/右游标不在时间轴最左/右端，左/右游标所在的时间调整为时间轴最左/右端所在的时间，数值轴不变，从而使曲线局部缩小，左/右游标位置分别置时间轴最左/右端。

²  当未选定矩形区域，左/右游标在时间轴最左/右端时，时间轴宽度调整为原来的二倍，保持中心位置不变，数值轴不变，从而使曲线局部缩小，左/右游标位置分别置时间轴最左/右端。

### 5.8.2    Word编辑控件方法

1、SHORT Open(string FilePath)

功能：控件在后台打开一个当前所要操作的word文档，之后的方法都是对这个文档进行操作。

参数：

       FilePath: 所要打开的文件的目录

返回值：

        0：打开成功

       -1：文件不存在或者格式不正确

       -2：未正确安装word

       -3：该文件当前被ShowWord函数打开，尚未关闭

注意：

在使用ShowWord函数在显示word文档，而尚未用CloseShowWord关闭时，不可使用Open函数在控件内部打开该文档； word软件当前打开的文档也不能使用Open函数在控件内部打开该文档。

软件处于运行态时，脚本中所涉及到的所有word文档都不能使用word软件打开，否则会发生word数据读写错误等。

2、SHORT OpenAsReadOnly(string FilePath)

功能：控件在后台打开一个当前所要操作的word文档，仅具有只读权限。

参数：

       FilePath: 所要打开的文件的目录

返回值：

        0：打开成功

       -1：文件不存在或者格式不正确

       -2：未正确安装word

3、Void SaveAs(string FilePath)

功能：将当前在后台打开的word文档进行保存

参数：

       FilePath：所要保存的文件目录

返回值：无

 

4、Void Close()

功能：关闭控件在后台打开的word文档

参数：无

返回值：无

5、BOOL  FilePathIsValid(string FilePath)

功能：用来判断指定目录的文件是否存在

参数：

       FilePath：文件目录

返回值：

       True：文件存在

       False：文件不存在或者格式不正确

6、SHORT SetText(int Table, int Row, int Col, string
Text)

功能：如果所要填充的单元格为空，则使用该方法进行文本的写入

参数：

       Table：所要写入的表格的序号，大于0

       Row：所要写入的行，大于0

       Col：所要写入的列，大于0

       Text：所要写入的文本

返回值：

        0：写入成功

       -1：控件当前未在后台打开任何文档

       -2：参数无效

7、string GetText(int Table, int Row, int Col)

功能：使用该方法读取单元格内所有内容

参数：

       Table：单元格所在的表格的序号，大于0

       Row：单元格所在的行，大于0

       Col：单元格所在的列，大于0

返回值：单元格内的所有文本（字符串），如果获得失败则返回空字符串

      

8、SHORT SetTextByBookmark(string Bookmark, string
Text)

功能：在指定书签位置写入文本

参数：

       Bookmark：word内的书签所对应的字符串，书签需先在word文档中手动添加

       Text：所要写入的文本

返回值：

        0：写入成功

       -1：控件当前未在后台打开任何文档

9、string GetTextEx(int Table, int Row, int Col, string
LeftMark, string RightMark)

功能：使用该方法获取单元格内字符串LeftMark与RightMark之间的文本

参数：

       Table：单元格所在的表格的序号，大于0

       Row：单元格所在的的行，大于0

       Col：单元格所在的的列，大于0

       LeftMark，RightMark：用来标记所要获得的内容的左右两侧的字符串，举例来说：

       1.单元格内为"出厂批号： 11 批"，若要返回"11 "，则LeftMark为"出厂批号："，RightMark为"批"。

       2.单元格内为："35 ℃"，若要返回"35"，则LeftMark为""，RightMark为"℃"。

       3.单元格内为："验收员：张红"，若要返回 "张红"，则LeftMark为"验收员："，RightMark为""。

若RightMark为""，则文本截取到整个单元格的末尾，示例如下：

若要获得"内容1"：LeftMark为"标签1：",
RightMark为" 标签2："

若要获得"内容2"：LeftMark为"标签2：",
RightMark为" 标签3："

若要获得"内容3"：LeftMark为"标签2：",
RightMark为""

获得的文本已去掉左右空格及回车

返回值：所要获取的指定字符串间的文本，已去掉字符串左右的空格

10、string GetTextEx2(int Table, int Row, int Col, string
LeftMark, string RightMark, int Order)

功能：使用该方法获取单元格内字符串LeftMark与RightMark之间的文本。

参数：

       Table：单元格所在的表格的序号，大于0

       Row：单元格所在的的行，大于0

       Col：单元格所在的的列，大于0

       LeftMark，RightMark：用来标记所要获得的内容的左右两侧的字符串，用法同GetTextEx

Order：表示LeftMark的序号，单元格中存在多个LeftMark相同的情况时，设定Order值可指定读取哪一个LeftMark后的内容。

返回值：所要获取的指定字符串间的文本，已去掉字符串左右的空格
