# 历史库导出工具使用手册

SCADA的数据集函数能够实现将历史库数据转存到关系库的功能，但这种方法需要用户输入大量的脚本，并且当转存数据较多或转存较频繁时，会消耗SCADA大量的线程资源。为此SCADA开发了一个独立的历史库到关系库的转存工具。本文档将介绍该工具的使用方法。

---

## 一、      启动历史库导出工具

找到HistoryDataToDB.exe可执行文件并双击该文件，弹出“HistoryDataToDB”对话框，如下图所示。

图1-1 历史库导出工具主界面

在图1-1-的对话框中，可以设置历史库数据导出的相关配置。历史库数据导出的配置主要包括以下几个方面：

1.        
要导出数据的历史服务器；

2.        
导出到的关系数据库；

3.        
映射关系的配置（映射表的创建和导出变量的选择）；

4.        
数据导出条件；

5.        
保存配置；

6.        
执行导出。

该文档将按照上面的几个方面来详细讲述历史库导出工具的使用方法和操作的注意事项，并在最后展示如何查看导出的数据。

---

## 二、      历史服务器配置

图1-2 历史服务器

上图的红色标示部分是关于历史服务器的配置。

**服务器类型：**要导出数据的历史服务器，有Scada和KingView两个选项。

**历史服务器：**当选择Scada历史服务器时，应填写历史服务器所在的计算机的IP地址；当选择KingView历史服务器时，则应输入工程文件的路径。

填写好历史服务器后，单击图1-1的“连接历史库”按钮连接配置的历史服务器。连接成功后站点列表会枚举出当前历史服务器下所有站点。常见的连接失败原因是：未启动Scada或KingView的运行系统。在这种情况下，会弹出如下图所示的对话框。

图1-3 历史库连接错误提示

这时，需要启动Scada或KingView的运行系统并重新进行历史库的连接。

**站点：**历史服务器下所有站点的站点名称。**注意：**此处列出的站点为工程中曾经用过的所有站点。因为历史库存储时会自动记录这些站点名，查询某时间段内的历史数据时也必须指定这段时间内使用的站点名，否则会查询失败。

**注意：**因为组态王API不支持变量数据类型的查询，所以当以组态王历史数据作为导出数据时，所有数据类型均会被当做float类型来对待。

## 三、      关系数据库配置

图1-4 关系数据库

**类型：**有SQL Server
2005和Oracle两个选项，根据实际情况进行选择。

**用户名：**登录关系数据库的用户名。

**密码：**登录关系数据库的密码。

**数据库：**要连接的数据库名称。

**服务器：**要连接的服务器名称，只在选择了SQL Server 2005数据库时，需要输入。

**目录：**该项参数暂时不可用，为今后升级功能预留。

完成关系库的配置后，单击图1-1的“连接关系库”按钮连接配置的数据库。若连接失败，系统会弹出连接失败的提示对话框，如下图所示。

图1-5 关系库连接错误提示

**注意：**单击“连接关系库”按钮后，无论在连接历史服务器时是否选择了站点，都必须重新选择历史服务器的站点。

---

## 四、      映射关系配置

**1.****创建映射表**

图1-6 创建映射表

输入表格名称并选择表格所存储的变量类型（Bool、Int和Float），单击“添加表”按钮，则会在指定数据库中自动创建表格，并且被创建的表格会自动添加到“数据表”列表中。**注意：**输入的表格名称不能与数据库中已存在的表重名，若重名，会提示用户重新输入表格名称。

创建表格的字段及类型：

|  |  |  |
| --- | --- | --- |
| 字段名 | 数据类型 | 备注 |
| TagName | nvarchar(256) | 变量名，格式：Station.Tag |
| Value | bit、int、float | 变量值，根据存储变量类型的不同会是不同的数据类型 |
| TimeStampKX | datetime | 时间戳，存储的是当前时区的时间 |
| QulityStamp | bigint | 质量戳，为一个十进制整数，详细信息请参阅KingSCADA相关帮助文档 |

 

**2.****添加导出变量**

创建好映射表之后，在“变量映射”中添加历史服务器中要导出的变量。

**变量列表：**列举了所选站点中所有有历史记录属性的变量，该列表支持使用Ctrl或Shift键配合鼠标完成多选，不支持鼠标框选。当鼠标点击的变量不是上面表格指定的类型时，变量无法选中。

**导出列表：**列举了要导出的变量，与变量列表一样，支持多选。

选中“变量列表”中的变量，单击按钮将变量从“变量列表”移到“导出列表”中；单击按钮将 “导出列表”中的变量移回“变量列表”，即每个变量只能被添加到一个数据表中。

图1-7 添加导出变量

**例如：**上图中的“导出列表”中变量，在下次重新打开HistoryDataToDB.exe或创建新的映射表时，将不会在变量列表中显示。

## 五、      数据导出条件配置

图1-8 数据导出条件

目前只支持“无条件导出”。第一次执行导出时会默认导出当前时间之前1小时之内的历史数据。工具会自动保存上一次导出的截止时间，并在下次导出时，以此为开始时间从历史服务器上导出数据。

目前支持“按时间间隔导出”，设置的导出时间间隔不得小于5分钟。

---

## 六、      保存配置

图1-9 功能操作按钮

在历史库导出工具的下方，提供了6个功能操作按钮。在完成上面所有的配置以后，单击“保存配置”按钮将历史服务器信息、数据库连接信息和映射关系信息保存到工具所在路径下的HistoryToRelation.db文件中。

重新启动工具，在连接历史库时会自动加载配置文件中的配置信息，并会自动将保存的变量配置信息与当前历史库中的变量做比较，如果发现错误会弹出如下图所示的对话框。

图1-10 加载配置文件错误

在上图的对话框中，用户可选择“过滤变量”和“清空配置”两种操作。

**“过滤变量”**：单击该按钮，工具会自动将不匹配的变量信息从配置中删除，保留合法的变量。此操作在变量多时可能会比较慢。

**“清空配置”**：单击该按钮，工具会清空以前配置的所有变量映射信息，并恢复成初始状态，在非法变量较多时比较实用。

---

## 七、      执行导出

单击图1-1的“执行”按钮，开始定时执行导出操作。单击“后台执行”按钮，则开始执行导出，并且工具缩小为任务栏图标。双击图标后显示主界面。

执行导出过程中，图1-1主界面的“执行状态”中会显示下次导出时间和现在时间，如下图所示。

图1-11 执行状态

执行导出过程中也会生成日志信息，并保存在工具所在路径的HistoryToRelation\_Log.db文件中，可使用文本编辑器查看此文件。

**注意：**不能在导出数据时查看日志文件，否则会引起工具异常。

---
